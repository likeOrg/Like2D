# Document / support scene composing vs. stacking

Three patterns:
 1. **Opaque stacking**: `like.pushScene(scene, false)` The scene stack is a state tracker. No scene cares about its parent or child. The parent is unloaded when pushed, and re-loaded when popped.
 2. **Scene composition**: `like.pushScene(new Parent(new Child))` Pass a child scene **with a known interface** into the parent constructor. The parent calls child callbacks accordingly, filtering the event stream. Examples: a UI overlay that interacts with a game. A level editor.
 3. **Overlay stacking**: `like.pushScene(scene, true)` The top scene has access to the bottom scene, but knows nothing about it except that it's a scene. Examples: Pause screen, gamepad mapping overlay.
