# Scene generator API
Instead of passing an instance of a scene, we are encouraged to pass
a function with signature `(like) => Scene`.

This simplifies our lifecycle: the load callback becomes redundant
with the constructor, and our types are no longer nullable.

We switch to storing scene instances as a function for consistency:
```
sceneDispatch(scene, like, 'load');
return scene;
```

This way, it's symmetrical. Backwards compat, but a nice new feature.

Our preexisting scenes will be switched to this architecture.

Scenes call `unload` when popped.

If our scene is disposed when pushed over, `load` should be called when it is popped. Otherwise not.
